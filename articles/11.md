# 11｜AI 輸出品質不穩？建立你的 prompt 模板庫

## 這篇解決什麼問題

同樣的任務，有時候 AI 產出很好，有時候很差。不是 AI 不穩定——是你每次給的 prompt 品質不一致。模板庫解決這個問題：把你最好的一次 prompt 固定下來，以後每次都至少 80 分。

## 我原本的做法 → 現在的做法

**原本：** 每次都臨時想 prompt。有時候狀態好，寫得詳細，AI 產出就好。有時候趕時間，隨便問，AI 產出就差。品質像抽獎。

**現在：** 常用場景都有模板。打開，填空，送出。品質穩定在 80 分以上。模板用了覺得不好就改，改完再存回去。持續迭代。

## 完整流程

### 步驟一：盤點你的高頻場景

```
請幫我盤點適合建立 prompt 模板的場景。

我的日常工作包括：
[描述你的工作內容]

我使用 AI 的場景包括：
[列出你常用 AI 做的事]

請幫我：
1. 列出使用頻率最高的 10 個場景
2. 按「頻率 × 品質波動度」排序（越高越值得做模板）
3. 建議前 5 個優先建立模板的場景
```

### 步驟二：把好的 prompt 轉成模板

```
以下是我用過效果最好的一次 prompt：

[貼上那次的 prompt]

請幫我把它轉成可重複使用的模板：
1. 把特定內容替換成 [填空欄位]
2. 加上「使用說明」（什麼時候用、怎麼填）
3. 標出「必填」和「選填」欄位
4. 加上一個填寫範例
```

### 步驟三：模板迭代

模板不是寫完就放著。每次用完，問自己：結果滿意嗎？哪裡可以更好？

```
我剛用了這個模板，但結果不太滿意：

模板：[貼上模板]
我填的內容：[貼上填寫內容]
AI 的輸出：[貼上輸出]
我不滿意的地方：[描述]

請幫我：
1. 分析問題出在模板還是我填的內容
2. 如果是模板的問題，給出修改版
3. 如果是填寫的問題，給出填寫建議
```

模板庫放在哪？Obsidian 最適合——可以搜尋、分類、跟知識庫連結。不用 Obsidian 也沒關係，放在你隨手拿得到的地方就好。

## 常見卡關點

**「模板太多管理不了」**
不要超過 20 個常用模板。如果超過了，說明你的分類太細。合併同類的，歸檔不常用的。

**「模板限制了創意」**
模板處理的是 80% 的重複部分。剩下 20% 你自由發揮。模板不是牢籠，是地板——讓你的起點更高。

**「不知道存在哪」**
不管你用什麼工具，關鍵是「一個地方，找得到」。最怕的是模板散落在五個 app 裡，用的時候找不到，最後又回到臨時想。
