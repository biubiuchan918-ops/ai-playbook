# 02｜Prompt 不是「下指令」，是「給脈絡」

## 這篇解決什麼問題

大多數人寫 prompt 像在命令機器：「幫我寫一封 email」。但 AI 不是搜尋引擎的升級版——它更像一個很聰明但什麼都不知道的新同事。你不是在「下指令」，你是在「交代背景」。

## 我原本的做法 → 現在的做法

**原本：** 花時間研究各種 prompt 技巧——角色設定、few-shot、chain-of-thought。把 prompt 當成一種「魔法咒語」，以為用對格式就能得到好結果。

**現在：** 不再想「怎麼寫 prompt」，而是想「我到底要什麼」。想清楚之後用白話跟 AI 說。技巧重要嗎？重要。但 90% 的情況，你把脈絡講清楚就已經夠好了。

大部分人用 AI 的方式，跟用 Google 沒有本質分別。打開對話框，丟一個問題進去，拿到一個答案，關掉。每次都是一張白紙，每次都要重新解釋你是誰、你要什麼。就像你每天上班都跟一個全新的實習生合作——他很聰明，但你每天都要從頭教起。

## 完整流程

### 脈絡三層法

**第一層：你是誰（角色脈絡）**
```
我是 [角色]，在 [公司/領域] 工作，
主要負責 [職責]。
```

**第二層：你要什麼（任務脈絡）**
```
我需要 [具體產出]，
用途是 [這東西要拿來做什麼]，
讀者/受眾是 [誰會看到這個]。
```

**第三層：你的標準（品質脈絡）**
```
好的版本應該 [描述理想狀態]，
要避免 [你不想看到的東西]，
參考風格像 [舉一個例子]。
```

### 完整範例

差的 prompt：
> 「幫我寫一篇 Threads 貼文。」

好的 prompt：
```
我是一個經營 Patreon 的內容創作者，主要寫 AI 工具的實戰應用。

我需要一篇 Threads 貼文，介紹我用 Claude 整理筆記的方法。
這篇的目的是讓讀者覺得「原來可以這樣用」，然後點進 Patreon 看完整版。

好的版本應該：
- 用第一人稱，像在跟朋友聊天
- 第一句就讓人想繼續讀
- 控制在 300 字以內
- 結尾有清楚的 CTA

要避免：
- 列點式（Threads 上讀起來太像教學文）
- 「AI 將改變世界」這類空話
```

兩個 prompt 的差距在哪？不是格式，是脈絡。後者告訴 AI：我是誰、我要什麼、我的標準是什麼。AI 有了這些，才能從「通用回答」跳到「為你量身定做的回答」。

### 如果你不知道自己要什麼怎麼辦？

用否定法。先讓 AI 出一版，然後跟它說「哪裡不對」：

```
這個版本太正式了，我要的感覺更像在跟朋友聊天。
開頭太平淡，我要那種讓人一秒被勾住的開場。
結尾的 CTA 太硬銷了，要自然一點。
```

很多時候你說不清楚「要什麼」，但看到不要的東西會馬上知道。這個迭代過程本身就在幫你釐清需求。

## 常見卡關點

**「每次都要打這麼多字很煩」**
這就是為什麼你需要模板庫。把你最常用的脈絡存成模板，下次直接填空。第 11 篇會詳細講怎麼建。

**「同樣的 prompt 結果每次不一樣」**
加上具體的格式要求和範例。越具體，AI 的「創意空間」越小，輸出就越穩定。如果你需要一致性，就把「好的範例」直接貼進 prompt。

**「我覺得我跟 AI 的關係就是問答，沒什麼不好啊」**
問答沒有錯。但問答是一次性的，協作是有累積的。如果你每次都從零開始，你永遠在 Level 1。這篇教的脈絡三層法是跳板——當你把脈絡寫好、存成模板、建成記憶系統，你的 AI 使用就從「問答」升級成了「協作」。
