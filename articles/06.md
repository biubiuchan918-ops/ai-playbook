# 06｜用 AI 準備一場演講：從大綱到逐字稿

## 這篇解決什麼問題

接到演講邀約的第一反應通常是焦慮。不是不知道要講什麼，是不知道怎麼把一堆想法組織成一場流暢的 30 分鐘演講。AI 特別擅長解決「結構」問題。

## 我原本的做法 → 現在的做法

**原本：** 打開空白簡報，邊想邊做。第一版太散，改。第二版太長，砍。第三版語氣不對，重寫。五次以上才定稿，每次至少花兩天。

**現在：** 先花 10 分鐘跟 AI 對話，釐清核心訊息。再花 15 分鐘讓 AI 排結構。最後 30 分鐘轉成口語化逐字稿。我只需要加入個人故事和臨場感，不用從零搭骨架。

我以前準備演講的方式其實很亂來——用好幾個 AI 工具拼拼湊湊，這個幫我寫大綱、那個幫我改措辭，結果素材散落一地，拼出來的東西反而沒有一條清楚的主線。現在有了整套 AI OS，情況完全不同：我的讀書筆記、過去寫過的文章、校準過的觀點全部整理好了，隨時可以調動。準備演講的時候，我不是從零開始找素材，而是從一個已經整理好的知識庫裡挑選和組合。

## 完整流程

### 步驟一：釐清核心訊息（10 分鐘）

```
我要準備一場演講，資訊如下：

場合：[什麼活動]
聽眾：[什麼背景的人，大約多少人]
時間：[幾分鐘]
主辦方希望我講：[他們給的主題或方向]

我自己最想傳達的一件事是：[一句話]

請幫我：
1. 用一句話定義這場演講的核心訊息
2. 從聽眾角度，列出他們來聽這場演講最想得到的 3 件事
3. 建議一個讓人印象深刻的開場方式
```

### 步驟二：建立結構（15 分鐘）

```
基於剛才的核心訊息，請幫我設計演講結構。

限制條件：
- 總時長 [X] 分鐘
- 我想包含 [N] 個個人故事/案例
- 結尾要有明確的 call to action

請用這個格式：
1. 開場（X 分鐘）：[做什麼]
2. 第一段（X 分鐘）：[主題] — [要達到的效果]
3. ...
4. 結尾（X 分鐘）：[做什麼]

每段之間請加上「過渡句」的建議。
```

### 步驟三：口語化逐字稿（30 分鐘）

```
請根據上面的結構，寫出口語化的逐字稿。

語氣要求：
- 像在跟朋友聊天，不是在唸論文
- 用短句，一句話不超過 20 個字
- 每 2-3 分鐘要有一個讓聽眾反應的點（笑點、提問、意外的數據）

標記格式：
[停頓] = 刻意停頓的地方
[看觀眾] = 要跟觀眾眼神交流的地方
[換投影片] = 切換投影片的時間點
```

## 常見卡關點

**逐字稿寫好但講起來不自然**
一定要大聲唸出來。眼睛看文字覺得通順的，嘴巴說出來可能很卡。唸不順的地方就改，沒有例外。

**時間抓不準**
中文演講大約 1 分鐘 250 字。用字數回推：30 分鐘的演講，逐字稿約 7,500 字。這個數字比大多數人以為的少——因為你需要留空間給停頓、互動、和觀眾笑的時間。

**太依賴投影片**
AI 幫你做的是「講稿」，投影片應該最後才做。先確定你不看投影片也能講，再用投影片輔助。順序反了，你會變成在唸投影片。
